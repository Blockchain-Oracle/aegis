# MCP client examples

This directory shows how to run **both** the official Splunkbase
**Splunk MCP Server** (app 7931, by Splunk LLC) and **SplunkGate MCP**
in the same Claude Desktop / Cursor / generic MCP-client session.

## Why both?

The two servers do **structurally different** jobs:

| Server | What it does | Tools (subset) |
|---|---|---|
| **`splunk-mcp-server`** | Connects an agent to **Splunk data**. The agent reads/queries the Splunk platform via a Splunk-issued token. | `splunk_run_query`, `splunk_search`, `splunk_get_indexes`. With SAIA installed: `generate_spl`, `explain_spl`, `optimize_spl`, `ask_splunk_question`. |
| **`splunkgate-mcp`** | Wraps **agent safety** around the agent's other tool calls. The agent asks SplunkGate *before* acting, *and* asks SplunkGate to log the verdict after acting. | `splunkgate_score_prompt_injection`, `splunkgate_judge_tool_call`, `splunkgate_check_output_leak`, `splunkgate_audit_trace`. |

The workflow you get for free with both running:

```
user message arrives
  ├─ agent calls splunkgate_score_prompt_injection(user_text)
  │   └─ if BLOCK: refuse, log, done
  ├─ agent reasons: "I need Splunk data"
  │   └─ agent calls splunk_run_query(...)  ← official Splunk MCP
  ├─ agent composes a response
  │   └─ agent calls splunkgate_check_output_leak(response_text)
  │       └─ if BLOCK: redact or refuse
  ├─ agent calls splunkgate_audit_trace(trace_id)
  └─ response surfaces to user
```

No proxy. No fork. No rebuild. Standard MCP client multi-server config —
both servers run side-by-side in the same JSON file.

## Files

### `claude-desktop-config.json`

Drop-in `claude_desktop_config.json` for Claude Desktop on macOS.
Replace `YOUR_SPLUNK_HOST` and `YOUR_SPLUNK_TOKEN` with your tenant values.
Path:

```
~/Library/Application Support/Claude/claude_desktop_config.json
```

For Cursor: copy the `mcpServers` block into `~/.cursor/mcp.json`.

## Prerequisites

### `splunk-mcp-server` (Splunkbase app 7931)

1. Install **Splunkbase app 7931** on your Splunk Search Head:
   <https://splunkbase.splunk.com/app/7931>
2. Add the `mcp_tool_execute` capability to your role.
3. Generate a token: Splunk Web → Settings → Tokens.
4. Optionally install **Splunkbase app 7245** (Splunk AI Assistant for SPL)
   to unlock `generate_spl` / `explain_spl` / `optimize_spl` /
   `ask_splunk_question` in the MCP server's tool list.

Reference: <https://help.splunk.com/en/splunk-cloud-platform/mcp-server-for-splunk-platform/1.1/configure-the-splunk-mcp-server>

### `splunkgate-mcp` (this repo)

1. Install in editable mode:
   ```bash
   uv pip install -e packages/splunkgate_mcp
   ```
2. (Optional) Set a Cisco AI Defense API key for live verdict calls:
   ```bash
   export SPLUNKGATE_AI_DEFENSE_MOCK=false
   export SPLUNKGATE_AI_DEFENSE_API_KEY=...
   ```
3. (Optional) Run the Hosted Models setup so verdict explanations are
   generated by a real LLM rather than the template fallback:
   ```bash
   bash scripts/setup_hosted_models.sh
   ```
